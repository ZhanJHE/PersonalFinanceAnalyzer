## 二、功能演示（5-6 分钟）

> 按 Tab 从左到右依次演示。服务器和客户端已提前启动。

### 1. 启动与登录（40 秒）

**操作**：
1. 展示两个终端：服务器运行中 + 客户端已启动
2. 点击「用户 → 登录」
3. 用测试账户 **张三 / 123456** 登录

**讲解要点**：
- 服务端输出日志：[Dev] 种子数据导入完成（20 条交易）
- 登录后状态栏显示"已登录：张三"，同步自动触发

### 2. 概览页 — Dashboard（40 秒）

**操作**：
1. 停留在「概览」Tab
2. 指一下三个卡片：本月收入 ¥16,000 / 本月支出 ¥6,480 / 结余 ¥9,520
3. 下面近期交易列表显示本月前 10 条

**讲解要点**：
- 数据来源：登录后从云端同步到本地 SQLite
- 卡片数据实时计算

### 3. 记录管理 — 增删改查（1 分钟）

**操作**：
1. 切换到「记录」Tab
2. **筛选**：选 2026-05-01 至 2026-05-31 → 点筛选 → 只显示 5 月数据
3. **搜索**：输入"购物"→ 点搜索 → 只显示购物相关记录
4. **新增**：类型=支出，类别=交通，金额=15，日期=今天，备注="共享单车"→ 点新增
5. **编辑**：双击刚才新增的记录 → 把金额改成 20 → 保存
6. **删除**：选中一条 → 点删除 → 确认

**讲解要点**：
- MVVM 架构：数据绑定 + RelayCommand
- 操作实时同步到本地 SQLite（登录时同步到云端）

### 4. 图表分析 — 净值折线图/饼图（1 分钟）

**操作**：
1. 切换到「分析」Tab
2. 默认展示**最近一个月** Tab：上方为收支净值**折线图**（每 2 天一个数据点，标注金额），下方为支出分类占比**饼图**（扇形标注百分比）
3. 点 **「最近三个月」** Tab → 切换到 90 天范围（每 6 天一个数据点），已预渲染切换零延迟
4. 点 **「最近半年」** Tab → 切换到 180 天范围（每 12 天一个数据点）
5. 三个 Tab 各自独立含折线图 + 饼图，图表底部显示数据范围

**讲解要点**：
- 单条净值线 + 0 轴分界：绿色=收入>支出，红色=支出>收入
- 固定 15 个数据点，自适应时间跨度（1月=每2天，3月=每6天，6月=每12天）
- 三种范围数据预计算、一次性渲染，Tab 切换零延迟
- SQLite DailyView 视图做日汇总，图表基于视图计算减少查询量
- ScottPlot 5.x 图表库，禁用鼠标交互（静态展示），完全依靠图例和标签

### 5. AI 智能分析（1 分钟）

**操作**：
1. 在**最近一个月** Tab 中点 **「获取 AI 分析建议」** 按钮 → 弹出 AI 报告窗口（Markdown 渲染）
2. 切换到**最近三个月** Tab → 点 **「获取 AI 分析建议」** → 展示季度分析报告
3. 切换到**最近半年** Tab → 点 **「获取 AI 分析建议」** → 展示半年总结报告

**讲解要点**：
- 客户端不直接调 DeepSeek → 经后端代理（API Key 在服务端）
- AI 报告以独立弹窗展示（MarkdownScrollViewer 渲染），不占页面空间
- 系统记录调用次数，免费用户每月 10 次，当前会员不限
- Prompt 分三类：1个月（月度总结）、3个月（季度分析）、6个月（半年总结），模板各不相同
- 未登录时弹窗询问是否前往登录

### 6. 预算管理（40 秒）

**操作**：
1. 切换到「预算」Tab
2. 选类别「购物」，预算上限设为 ¥2,000，点设置
3. 切回「记录」Tab，新增一笔购物支出（金额随便填）
4. 因为 6 月已有 ¥3,200 购物支出，会**立即弹窗**"超预算提醒"

**讲解要点**：
- 每次新增/编辑支出后自动检查当月预算
- 预算按类别+年月存储，可随时修改
- 每个预算项使用类别颜色作底色美化（如餐饮=粉红、购物=蓝色）

### 7. 数据导入导出（40 秒）

**操作**：
1. **导出 Excel**：文件 → 导出 Excel → 保存 → 展示生成的 .xlsx 文件（含合计公式）
2. **微信账单导入**（口头带过）：文件 → 微信导入指南 → 展示引导窗口

**讲解要点**：
- Excel 用 ClosedXML 库生成，带表头样式和 SUM 公式
- 微信账单解析器自动识别格式，关键词匹配归类别

### 8. 云端同步（30 秒）

**操作**：
1. 点状态栏「同步」按钮 → 状态显示"已同步 ✓"
2. 展示两次启动间数据不丢失

**讲解要点**：
- JWT 认证 + HTTPS + Rate Limiting
- SHA256 哈希快速比对，哈希一致跳过同步
- 哈希不一致时三选一窗口：保留本地 / 保留云端 / 逐条处理
- **冲突处理**：同一笔记录两边都修改过时，弹出逐条选择窗口
- **退出同步**：关闭程序时询问是否同步，防止未保存数据流失
- 离线仍可用本地模式

## 三、技术讲解（2-3 分钟）

> 建议配合 README 或架构图讲解，不需要操作

### 1. 整体架构

```
WPF 客户端 ←→ ASP.NET Core 后端
  ↓               ↓
本地 SQLite      云端 SQLite
```

- **为什么两层**：比三层的数据库服务器更简单，演示只需一台机器
- **为什么数据分两份**：本地离线可用，云端多设备同步

### 2. 安全措施

| 措施 | 说明 |
|------|------|
| BCrypt | 密码哈希存储，不存明文 |
| JWT Token | 登录后签发，过期自动失效 |
| HTTPS | 通信加密 |
| Rate Limiting | 登录 10次/分钟，注册 5次/分钟 |
| DPAPI | Token 加密存本地 |

### 3. MVVM 架构

```
View (XAML) ← 数据绑定 → ViewModel ← 调用 → Service → SQLite/HTTP
```

- CommunityToolkit.Mvvm 源码生成器自动生成命令和属性通知
- Service 层接口抽象，支持替换实现（离线/在线模式切换）

### 4. 使用的 NuGet 包

| 包 | 用途 |
|----|------|
| CommunityToolkit.Mvvm | MVVM 框架 |
| ScottPlot.WPF | 图表可视化 |
| Microsoft.Data.Sqlite | 本地 SQLite |
| ClosedXML | Excel 导出 |
| BCrypt.Net-Next | 密码哈希 |
| MySql.EntityFrameworkCore | MySQL 开发支持 |
