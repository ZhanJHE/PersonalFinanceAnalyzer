# 个人收支趋势分析器 — 项目说明文档

> 本文档描述项目的实际实现状态，供课程评审参考。

---

## 📌 项目概述

开发一个 **WPF 桌面应用 + ASP.NET Core 后端服务**，实现个人收支的记录、分类、趋势分析与可视化，并通过服务器代理调用 DeepSeek API 提供智能分析建议。

**核心特性**：
- 离线可用：本地 SQLite 存储，不依赖网络
- 在线同步：登录后数据自动同步到云端
- AI 分析：通过后端代理调用 DeepSeek，客户端不直接接触 API Key

## 📐 架构设计说明

### 为什么选用两层架构？

```
WPF 客户端 ←→ ASP.NET Core 后端
     ↓               ↓
  本地 DB          云端 DB
```

**三层架构（客户端 + 后端 + 数据库服务器）**对于课程项目过于复杂。当前的两层架构（客户端 + 后端，各自使用 SQLite）足够实现核心需求，且部署简单——只需一台机器即可运行演示。

### 为什么 AI 要经后端代理？

1. **安全**：DeepSeek API Key 存放在服务端，用户拿不到
2. **限额控制**：服务端统一统计使用次数
3. **可扩展**：未来更换 AI 模型只需改服务端，客户端无感知

### 为什么要做云同步？

虽然这是一个以本地为主的桌面应用，但增加云同步后：
- 用户可以在多台电脑上使用同一份数据
- 导出微信账单→导入电脑→云端保存→另一台电脑自动同步
- 数据不因电脑损坏而丢失

## 🚀 快速开始

### 1. 安装依赖

```bash
# 信任 HTTPS 开发证书
dotnet dev-certs https --trust

# 进入项目目录
cd PersonalFinanceAnalyzer
```

### 2. 配置 API Key

编辑 `PersonalFinanceAnalyzer.Server/appsettings.json`：

```json
"AiApi": {
    "ApiKey": "你的DeepSeek API密钥",
    "BaseUrl": "https://gateway.ai.vercel.app/v1/chat/completions",
    "Model": "deepseek/deepseek-chat"
}
```

### 3. 启动服务端

```bash
cd PersonalFinanceAnalyzer.Server
dotnet run
# HTTPS: https://localhost:5001
# HTTP:  http://localhost:5000
```

### 4. 启动客户端

```bash
cd PersonalFinanceAnalyzer
dotnet run
```

---

## ⚙️ 注意事项

1. **首次运行**会提示信任 HTTPS 证书，选择「是」
2. **服务器必须先启动**，客户端才能登录和同步
3. **离线模式**下仍可使用全部本地功能（新增/编辑/删除记录），登录后自动同步
4. **AI 分析需要登录**后方可使用，免费账户每月 10 次额度
5. `finance.db` 和 `finance_server.db` 在首次运行时自动创建
